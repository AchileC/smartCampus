\hypertarget{md_readme_autotoc_md4}{}\doxysubsubsection{\texorpdfstring{1. Forker le modèle de stack}{1. Forker le modèle de stack}}\label{md_readme_autotoc_md4}
{\bfseries{UN.\+E SEUL.\+E}} des développeuses/développeurs de votre équipe va {\bfseries{fork}} le présent dépôt, pour en créer un nouveau, dans le groupe correspondant à votre équipe \+: ~\newline
 {\itshape Par exemple pour l\textquotesingle{}équipe 1 du groupe de TP K1, le groupe est \+:} {\ttfamily 2024-\/2025-\/BUT-\/\+INFO2-\/\+A-\/\+SAE34/\+K1/\+K11}

{\bfseries{Remarque}} \+: \texorpdfstring{$>$}{>}Il n\textquotesingle{}est pas nécessaire de conserver le lien avec le modèle de stack, vous pouvez donc aller dans ~\newline
 \begin{quote}
Settings \texorpdfstring{$>$}{>} General \texorpdfstring{$>$}{>} Advanced (dans Gitlab) pour supprimer le "{}\+Fork relationship"{} de votre projet \end{quote}
\hypertarget{md_readme_autotoc_md5}{}\doxysubsubsection{\texorpdfstring{2. Cloner la stack du projet}{2. Cloner la stack du projet}}\label{md_readme_autotoc_md5}
Le membre de l\textquotesingle{}équipe qui a réalisé le fork, doit cloner ce dépôt sur son poste de travail

⚠️ {\bfseries{Si vous êtes sous Linux}} ~\newline
 \begin{quote}
Avant de démarrer la stack, il faut renseigner les variables qui se trouvent dans le fichier {\ttfamily .env} à la racine du dépôt ~\newline
 Vous pouvez obtenir l\textquotesingle{}id de votre user (et de son groupe) en lançant la commande {\ttfamily id -\/u \$\{USER\}} dans un terminal \end{quote}

À ce stade, les services {\ttfamily sfapp}, {\ttfamily database} et {\ttfamily nginx} sont créés et démarrés, autrement dit fonctionnels, alors \+:
\begin{DoxyItemize}
\item on fait {\ttfamily commit} et {\ttfamily push} pour partager avec les autres membres de l\textquotesingle{}équipe
\item on déclare tout les membres de l\textquotesingle{}équipe dans le dépôt du projet avec le rôle {\ttfamily Developer} (si ce n\textquotesingle{}est pas déjà fait \+:-\/))
\item chaque membre de l\textquotesingle{}équipe peut alors
\begin{DoxyItemize}
\item cloner ce nouveau dépôt sur son poste de travail
\item démarrer toute la stack docker du projet 
\end{DoxyItemize}
